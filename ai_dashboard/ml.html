How to Train the Model
Random Forest · scikit-learn
1
Navigate to backend directory
Open a terminal in the project's backend folder.
cd ai_dashboard/backend
2
Train with synthetic data
Generates 6,000 synthetic samples based on clinical tremor frequency profiles and trains the RF model.
python -m ml.train_model
3
Augment with real data (optional)
Loads your stored sessions from profiles.db and combines them with synthetic data. Real samples are weighted 3× more.
python -m ml.train_model --augment-from-db
4
Customize training
Adjust the number of trees and samples per class for better accuracy.
python -m ml.train_model \
--n-per-class 3000 \
--n-estimators 200 \
--max-depth 15
5
Inspect training data
Export the training data as a CSV to review in Excel or a notebook.
python -m ml.train_model --export-csv
How It Works

The model takes 7 features from each 2.56s ESP32 window:

b1 (4–6 Hz)
b2 (6–8 Hz)
b3 (8–12 Hz)
total_power
meanNorm
dom_ratio
spectral_centroid (most important — 23%)

150 decision trees each vote independently. The majority vote becomes the final prediction.

The model classifies into 4 classes:

No Tremor Parkinsonian Essential Physiological
Accuracy
94.5% test accuracy · 100% no-tremor detection · ~92% per-subtype
